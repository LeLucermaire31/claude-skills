# /workflow — 启动工作流

收到用户项目描述后，严格按 SKILL.md 执行：

1. **Phase 0**: 分析需求，判定类型 A/B/C，评估复杂度，列出 Skill 注入建议
2. **人工确认**: 展示分析结果，等待用户确认/调整
3. **Phase 1**: 按选定路径执行（A 分阶段把控 / B 自动执行 / C 混合模式）
4. **终审 + 收尾**: 验证通过后交付

全程 caveman full 模式。
